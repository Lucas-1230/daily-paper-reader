## 摘要
安全代理的评估通常衡量在充足推理预算下的峰值攻击能力，重点关注漏洞发现、漏洞利用开发、渗透测试和CTF完成情况。此类衡量方法有用但不全面：在实战安全运营中，每个推理步骤、工具调用、遥测查询和富化请求都消耗预算。我们通过成本-成功透镜，在攻击性Cybench挑战和防御性Splunk BOTS v1调查挑战上评估语言模型安全代理。我们不仅报告最佳情况下的成功率，而是在固定成本水平上比较模型，并按推理花费和工具花费分解性能。结果显示了红队和蓝队任务的不同扩展机制。攻击性CTF性能随着额外测试时计算而提升，且扩展的开源权重模型可以在保持成本竞争力的同时接近前沿专有系统。防御性SOC调查并未以同样方式扩展：成功更依赖于规范的工具使用、遥测导航和选择性富化，而不仅仅是原始推理预算。我们认为安全代理的基准测试应衡量经济效率和运营适用性以及任务成功率。成本感知的、原生SOC评估能更清晰地展示哪些模型在当今具有实际用途，以及防御性代理仍需改进的地方。我们提供了一个交互式网站展示结果：https://evals.frontier.security。

## Abstract
Security-agent evaluations commonly measure peak offensive capability under generous inference budgets, emphasizing vulnerability discovery, exploit development, penetration testing, and CTF completion. Such measurements are useful but incomplete: in operational security, every reasoning step, tool call, telemetry query, and enrichment request consumes budget. We evaluate language-model security agents through this cost-success lens on offensive Cybench challenges and defensive Splunk BOTS v1 investigation challenges. Instead of reporting only best-case success, we compare models at fixed cost levels and decompose performance by inference spend and tool spend. Our results show distinct scalingregimes for red- and blue-team tasks. Offensive CTF performance improves with additional test-time compute, and scaled open-weight models can approach frontier proprietary systems while remaining cost-competitive. Defensive SOC investigation does not scale in the same way: success depends more heavily on disciplined tool use, telemetry navigation, and selective enrichment than on raw reasoning budget alone. We argue that security-agent benchmarks should measure economic efficiency and operational fit alongside task success. Cost-aware, SOC-native evaluations provide a clearer picture of which models are practically useful today and where defensive agents still need to improve. We present an interactive website with our results https://evals.frontier.security.

*   **核心思想：成本-成功透镜**

*   **关键技术细节与流程**：

    *   **成本核算分解**：将总花费明确分解为两部分：
        *   **推理花费**：模型调用产生的 token 费用，从 API 账单或 OpenAI/OpenRouter 的费率与 token 计数重建。

*   **对比方法（模型）**：

    *   **成本效率**：总花费（推理+付费工具）、单次解决成本、每 1000 分成本等。
    *   **工具调用量**：非提交工具的总调用次数，用于关联模型行为。
    *   **拒绝率**：模型因内容政策拒绝回答的样本占比。
*   **防污染控制**：
    *   针对公开且老旧的 BOTS v1 数据集，专门设计了一个 **“无工具”控制组**。该组模型不调用任何工具，仅凭问题和预置上下文回答，以检验模型是否从训练数据中直接记忆了答案。

### 6. 论文的主要结论与发现

*   **攻击与防御任务有不同的扩展规律**：
    *   **进攻 (Cybench)**：性能随测试时计算预算的增加而**持续提升**。例如，当预算从 0.80 提升到 2.10 美元时，Claude Opus 4.8 的成功率从 55.6% 跃升至 74.4%，显示出大量未开发的潜力。扩展的开源模型（如 DeepSeek v4 Flash）可以在成本上具备竞争力。
    *   **防御 (BOTS v1)**：SOC 调查任务**不随原始推理预算简单扩展**。更高的预算和更多的工具调用并不保证更好的结果。例如，DeepSeek v4 Flash 将预算翻倍，分数仅从 73.0% 微升至 73.9%。防御成功更依赖于**纪律严明的工具使用、遥测导航和选择性富化**。
*   **防御性智能体的逆效率现象**：最好的防御模型（Claude Opus 4.8）用**最少的工具调用（603次）** 取得了最高分（93.9%）；而表现较差的模型却产生了数千次调用。这表明，在防御任务中，高质量的少量查询远胜于低质量的大规模探索。
*   **公共基准的污染风险**：无工具控制组的实验表明，仅凭记忆，Claude Opus 4.8 就能在 BOTS v1 上得到 74.8% 的分数。这证明，**对于公开的 SOC 基准，必须进行去污染检查，否则其绝对分数不能直接视为真实调查能力的证据。**
*   **政策拒绝的巨大影响**：在攻击性任务中，模型拒绝回答的比例是能力表现的一个关键因素，GPT-5.6 和 Fable 的多个高安全性版本因高拒绝率而获得了极低的分数。

### 7. 优点

*   **视角创新**：首次系统性地将**成本**作为评估安全智能体的核心维度，将经济效率置于与峰值能力同等重要的位置。
*   **方法严谨**：
    *   **成本分解清晰**，明确区分了推理成本和工具成本。
    *   引入了**回溯性配对分析**，在控制变量的前提下科学地衡量预算扩展的收益。
    *   为公开防御基准设置了**严格的防污染控制组**，这在类似研究中很少见，极大地增强了结论的可信度。
*   **发现深刻**：揭示了红、蓝队任务截然不同的扩展规律，并提出了“防御任务重质量而非数量”、“工具调用量无法代表防御能力”等具有实践指导意义的深刻见解。
*   **实践导向强**：提供的结果（如成本/分、单次解决成本）和结论直接为安全运营团队在选型和预算规划上提供了决策依据。

### 8. 不足与局限

*   **防御任务的代表性有限**：仅使用了 BOTS v1 这一个公开的、相对老旧的数据集。虽然论文指出了这个问题并做了控制，但其结论能否推广到更复杂、更多样的 BOTS v2/v3 或其他真实 SOC 环境，尚待验证。
*   **实验设计非全因子化**：作者承认其实验是观察性的，而非一个将所有模型在所有预算水平下均测的全因子随机化矩阵，可能遗漏某些模型在不同预算下的最佳表现。
*   **自然语言决策与限制**：智能体本身不了解成本或 token 预算，而是由外部中断机制控制。一个有预算意识的智能体可能会采取不同的策略，其行为模式有待探索。
*   **工具成本定价的静态性**：外部工具的定价是固定的，没有考虑不同API套餐、企业授权或动态定价等真实世界的复杂性。
*   **潜在偏差**：模型性能受 API 接口、提供商默认设置、提示缓存命中率等外部因素影响，难以完全控制。BOTS v1 中由于消息数量限制（250条）可能导致部分高调用量的模型（如DeepSeek）表现受到挤压。
