## 摘要
大型语言模型赋予了强大的代码补全系统，能够通过预测后续代码行来辅助开发者。然而，这些模型仍然容易受到后门攻击，恶意微调数据会秘密植入不安全行为。尽管防御技术有所进步，但自适应且复杂的后门攻击仍能逃避检测和缓解。我们提出 CodeTracer，一个取证框架，将恶意代码补全追溯到造成该行为的后门微调数据。在现实的部署后约束下运行，CodeTracer 仅依赖于微调语料库和已报告的错误补全事件。它从受损输出中提取结构化的行为指纹，将搜索范围缩小到语义相关的代码样本，并采用基于大语言模型的推理将不安全逻辑归因于特定的后门数据。在三个代表性漏洞案例和十种后门攻击以及十六个竞争基线上的广泛评估表明，CodeTracer 始终能实现高取证准确性、低误识别率，并对自适应攻击具有强大的鲁棒性。

## Abstract
Large language models have enabled powerful code completion systems that assist developers by predicting subsequent lines of code. However, these models remain vulnerable to backdoor attacks, where malicious fine-tuning data covertly implants unsafe behaviors. Despite advances in defensive techniques, adaptive and sophisticated backdoor attacks still evade detection and mitigation. We present CodeTracer, a forensic framework that traces malicious code completions back to the backdoor fine-tuning data responsible for them. Operating under realistic post-deployment constraints, CodeTracer relies solely on the fine-tuning corpus and the reported miscompletion event. It extracts a structured behavioral fingerprint from the compromised output, narrows the search to semantically relevant code samples, and employs LLM-based reasoning to attribute unsafe logic to specific backdoor data. Extensive evaluations across three representative vulnerability cases and ten backdoor attacks, along with sixteen competitive baselines, demonstrate that CodeTracer consistently achieves high forensic accuracy, low false identification rates, and strong robustness against adaptive attacks.

- **研究背景与动机**：基于大语言模型的代码补全系统已广泛用于开发加速，但微调阶段容易被植入后门（backdoor）。攻击者可在微调数据中加入隐蔽的触发-恶意载荷对，使模型在特定上下文中生成不安全代码（如禁用证书验证、绑定所有网络接口等），而常规行为正常，导致严重安全威胁。
- **现存问题**：现有防御（静态分析、异常检测）对高级或自适应后门效果不佳，且几乎未见反向追溯责任方的研究。传统梯度溯源法因大模型部署环境不保留梯度而失效；实例级或基于聚类的取证方法也难以适应代码后门的隐蔽性。
- **核心技术问题**：给定一个已经发生的恶意补全事件（包括提示和模型生成的恶意代码），如何在不访问梯度、不基于攻击者特定知识的情况下，从大规模微调数据中找出具体哪些训练样本导致该后门行为，实现**事后取证与责任定位**。

### 2. 论文提出的方法论
#### 核心思想
CodeTracer 将恶意补全看作中毒样本行为模式的“继承”，通过提取**结构化行为指纹**并在微调语料中进行语义聚焦与LLM推理，实现无梯度、攻击者无关的后门数据溯源。

#### 关键技术细节（三阶段管道）
1. **指纹提取（Fingerprint Extraction）**  
   - 使用外部LLM分析恶意补全事件（提示+补全代码），生成JSON格式的结构化指纹，包含字段如：漏洞类型、规范化模板、规范化语义、等价变体、攻击转换方法、原始代码片段等。  
   - 该指纹保留行为本质，忽略表面语法噪声，作为恶意行为的紧凑签名。

2. **取证范围缩小（Scope Narrowing）**  
   - 从指纹中提取原始恶意代码片段 `ĉ`，用预训练代码编码器（如UniXcoder）将其嵌入为向量。  
   - 将每个微调样本 `d` 切分为函数级片段 `d_i`，计算与 `ĉ` 的最大余弦相似度作为样本得分：  
     `H(ĉ, d) = max_{1≤i≤n} sim( fθ(ĉ), fθ(d_i) )`  
   - 根据分数取 top-K 个候选项组成取证范围 `S`，大大压缩后续分析规模。

3. **取证归因分析（Attribution Analysis）**  
   - 对候选集 `S` 中每个样本，将指纹与其一起输入 LLM，判断该样本是否实现相同的不安全行为。若回答包含“[Label: Yes]”，则该样本被标记为后门数据。

整体实现见论文算法1，以上步骤以外部 LLM 为核心，无需任何攻击者先验信息。

### 3. 实验设计
- **数据集**：从 GitHub 爬取2017–2022年 Python 仓库（100+星），得到 1,080,606 个源文件。其中包含触发环境的文件用于生成中毒样本，其余作为干净数据。最终微调集由 **80,000 个干净样本** 和若干后门样本混合（默认每种攻击生成20个中毒样本，TROJANPUZZLE因变体需140个）。
- **目标模型**：CodeGen‑Multi 自回归代码生成模型，按标准流程进行微调。
- **漏洞场景**：三类代表性漏洞负载——`jinja2`（模板注入）、`requests`（禁用证书验证）、`socket`（绑定0.0.0.0）。
- **攻击方法**：8种默认后门攻击（SIMPLE、COVERT、TROJANPUZZLE、CODEBREAKER‑SA、CB‑GPT、CB‑ChatGPT、BadCode、CodePoisoner）以及2种自适应攻击（嵌入扰动、对抗提示）用于鲁棒性测试。
- **对比基线**：共 **16 种方法**，涵盖中毒取证与归因的主要家族：All‑at‑Once、Step‑by‑Step、Binary Search、Context‑Cite、Self‑Citation、PoiF、RAGForensics、TracLLM、AttnTrace、RAGOrigin 等。
- **评价指标**：假阴性率（FNR）、假阳性率（FPR）、检测准确率（DACC）、攻击成功率（ASR）及运行时间/金钱成本。

### 6. 论文的主要结论与发现
- CodeTracer 在所有漏洞场景和攻击类型上均取得 **显著低于0.03的FNR**，且 FPR 和 DACC 均接近完美，远优于所有基线方法（基线FNR多高于0.5）。

- 框架具有**高计算效率**（单次溯源约47秒）和**极低金钱成本**，且对 top‑K、中毒数量、触发器类型不敏感。

- 扩展至百万级训练样本时，CodeTracer 仍持续有效。
- 现有基于静态分析、异常检测或软件故障定位的方法在后门代码背景下表现不佳，凸显 CodeTracer 的独特价值。

### 7. 优点

### 8. 不足与局限

- **可能受虚假报告影响**：如攻击者伪造误报事件，可能干扰溯源流程；论文建议引入人工审核，但未提供自动化甄别机制。
- **攻击场景的限定**：主要针对 Python 代码与三类漏洞场景；对其他编程语言或更多样漏洞类型的泛化性有待验证。
- **多跳攻击与复杂交互**：虽然实验覆盖了多跳场景，但极端复杂的代码依赖链可能仍有挑战。
