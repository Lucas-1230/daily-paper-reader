## 摘要
基于代理模型(ABM)依赖于简单、明确且可复现的规则进行个体决策，而复杂的集体行为则从代理间的交互中涌现。大语言模型(LLM)的最新进展使得用基于LLM的代理能力替换、丰富或扰动这些规则具有吸引力。然而，这引发了一个方法性问题：引入LLM驱动的决策如何影响ABM仿真的可靠性、计算成本和行为？我们使用流行的Python ABM库Mesa模型，并通过统计模型检验进行研究。基于Mesa与统计模型检验工具MultiVeStA的集成，我们扩展了经典的谢林隔离模型，采用混合群体：普通代理使用标准符号规则对邻居进行分类，而一个代理通过工具调用将此任务委托给LLM。启用LLM的代理接收邻居的自然语言描述，并调用工具来累加相似/不同邻居的计数器；这些计数器根据原始谢林动力学决定其满意度。这提供了一个最小但受控的设置，可以在其他方面标准的ABM中研究基于LLM决策的语义、操作和计算行为。我们报告了使用本地部署的不同规模LLM的初步实验，结果显示较小模型可能在简单的语义分类实验中失败，或在重复工具调用生成过程中变得操作上不可用，而经过测试的较大模型通过了这些初步检查。我们讨论了统计模型检验如何估计经典ABM可观测值，并量化将代理LLM组件引入仿真模型的影响。

## Abstract
Agent-based models (ABMs) rely on simple, explicit and reproducible rules for individual decision making, while complex collective behavior emerges from interactions among agents. Recent advances in large language models (LLMs) make it tempting to replace, enrich, or perturb these rules with LLM-based agentic capabilities. However, this raises a methodological question: how does introducing LLM-driven decisions affect the reliability, computational cost, and behavior of ABM simulations? We investigate this for Mesa ABM models, a popular Python library for ABMs, analyzed by statistical model checking. Building on Mesa's integration with the statistical model checker MultiVeStA, we extend the classical Schelling segregation model with a hybrid population: ordinary agents classify neighbors using the standard symbolic rule, while one agent delegates this task to an LLM through tool calls. The LLM-enabled agent receives natural-language descriptions of neighboring agents and invokes tools that increment counters of similar/different neighbors; these counters determine its happiness according to the original Schelling dynamics. This provides a minimal but controlled setting where the semantic, operational, and computational behavior of LLM-based decisions can be studied inside an otherwise standard ABM. We report preliminary experiments with locally served LLMs of different sizes, showing that smaller models may fail simple semantic classification experiments or become operationally unusable during repeated tool-call generation, while larger tested models pass these preliminary checks. We discuss how statistical model checking can estimate classical ABM observables and quantify the impact of introducing agentic LLM components into simulation models.

## 1. 论文核心问题与研究背景

- **研究动机**：传统基于代理建模（ABM）中的个体决策规则是透明、可复现的符号化规则，但复杂集体行为涌现。大语言模型（LLM）虽可赋予代理更灵活的语言理解与决策能力，却在可靠性、计算成本与行为影响上引入未知风险。
- **核心问题**：在标准的ABM（Mesa‑谢林隔离模型）中，用一个LLM驱动的代理替换单个局部分类决策，会如何影响仿真的（i）语义正确性/鲁棒性、（ii）系统动态行为、（iii）运行开销？是否存在使仿真结果失控或统计显著偏离原模型的可能？
- **整体意义**：为“智能代理化ABM”（Agentic ABM）提供了一次受控的可行性分析，并展示了统计模型检验（SMC）如何系统量化这种集成的效果。

## 2. 方法论

- **模型扩展思路**：不改动谢林模型的全局调度与移动规则，仅将一名代理的邻居分类逻辑从符号类型比较改为LLM工具调用。
  - 每个代理除符号类型外还携带自然语言自述（如“Sky is beautiful”/“Fire is beautiful”）。
  - LLM代理接收所有邻居的自述列表，通过调用两个绑定的工具 `increment_similar_count` 和 `increment_different_count` 累加计数器，此后的满意度计算与移动完全复用原代码。
- **LLM部署与接口**：使用Ollama本地服务，模型统一为Qwen3.5系列，温度设为0以降低随机性。工具调用接口基于LangChain的`bind_tools`实现，LLM仅能生成工具调用，不能直接操控网格或代理状态。
- **统计模型检验（SMC）**：利用黑箱SMC工具MultiVeStA与Mesa的已有集成，执行瞬态分析：
  - 估计每个仿真步骤的“满意代理比例”期望值，并提供95%置信区间。
  - 通过点对点t检验比较原始模型与LLM扩展模型的动态是否统计显著不同。
  - 同时记录每仿真步骤的LLM调用次数与总耗时。
- **离线鲁棒性预检**：先对LLM代理单独测试四种语义难度递增的消息（从直接偏好表达到多个邻居描述的工具调用），记录每次响应的正确性（正确/语义错误/数量错误/卡死）与耗时。

## 3. 实验设计

- **基础环境与参数**：谢林模型，20×20网格，同质性阈值0.7，密度0.8，少数群体比例0.2。
- **LLM模型对比**：Qwen3.5的0.8b、2b、4b、9b四个尺寸，均本地部署。
- **实验分两阶段**：
  1. **鲁棒性测试**：
     - 四种消息难度M1–M4（从单句显式偏好到含3个邻居描述并要求生成对应数量的工具调用）。
     - 对每个模型，分别在“蓝队”和“红队”角色下各执行3次，共约6次/消息/模型。
     - 定性分类正确/语义错误/工具数量错误/卡死，记录平均推理时长。
  2. **全ABM仿真与SMC对比**：
     - 仅使用通过了鲁棒性测试的模型（最终选用`qwen3.5:4b`，兼顾正确性与速度）。
     - 仅插入一名LLM代理，其余全为符号代理。
     - SMC要求估计的置信区间宽度≤0.1，置信度95%，执行20次独立仿真，每次仿真60步（每步LLM代理会调用一次LLM，当邻居数>0时可能会产生多个工具调用）。
     - 比较原始模型与LLM扩展模型的满意代理比例曲线，并进行t检验。
- **对比基准**：无传统机器学习基线，对比对象是纯符号规则的原始谢林模型。不同尺寸LLM之间互为参照。

## 4. 资源与算力

- **鲁棒性测试**：在Apple M4（24GB统一内存）上完成，仅推理，无训练。
- **全仿真SMC**：在NVIDIA DGX Sparx（120GB统一内存）上执行。
  - 20次仿真，每次61次LLM调用（60步+初始），LLM扩展模型总耗时为**8917秒**（约2.5小时）。
  - 原始符号模型仅需**2秒**。
  - 每次LLM推理平均约7.3秒，开销巨大。
  - 未报告GPU型号或是否利用GPU加速，但Ollama本地服务通常可调用GPU，推理速度受限于模型大小和硬件。

- **实验组数**：
  - 鲁棒性测试覆盖4个模型×4种消息×2种角色×3次重复 ≈ 96次（部分模型卡死导致缺失）。

  - 鲁棒性测试的次数足以发现小模型的一致性问题（如0.8b在M1红队即卡死，2b在M2红队全部错误等），结果清晰。

## 6. 主要结论与发现

- **小模型不可靠**：
  - `qwen3.5:0.8b` 在简单显式消息下即出现卡死，在间接表达中始终选出对立工具，无法用于ABM。
  - `qwen3.5:2b` 在间接消息中系统性地错误分类（如所有红队代理消息被当作相似），且在多邻居工具调用任务上完全卡死。
- **较大模型通过预检**：`qwen3.5:4b` 和 `9b` 在所有消息条件下均能正确生成所需类型和数量的工具调用，具备作为ABM组件的资格。
- **系统动态无显著改变**：加入一名LLM代理后，满意代理比例的瞬态演化曲线与原模型在统计上不可区分（t检验均未拒绝相等假设）。
- **计算开销极高**：LLM扩展导致仿真时间从2秒飙升至约2.5小时，主要原因在于每步多次LLM推理（平均每次约7秒）。
- **方法论维度的结论**：统计模型检验能有效估计ABM指标，并给出可控置信区间，适合评估LLM集成的影响；离线鲁棒性预检是安全嵌入LLM的必要步骤。

## 7. 优点

- **最小侵入设计**：仅替换一个代理的一条局部规则，其余完全不变，提供了高度受控的实验环境，可隔离LLM的影响。
- **明确的工具调用抽象**：利用LangChain工具绑定将LLM的输出限定为预定义的计数操作，避免了LLM直接操纵状态的风险，便于出错溯源。
- **分阶段验证路径**：先离线单元测试LLM的语义理解与工具调用能力，再上线全仿真SMC分析，流程合理且具有工程参考价值。
- **统计严谨性**：采用SMC自动确定仿真次数，提供置信区间和t检验，将比较从“肉眼观察”提升到统计推断。
- **量化开销**：记录了LLM调用的总耗时与单次推理时间，为后续计算资源评估提供基准。

## 8. 不足与局限

- **场景过于简单**：仅在经典谢林模型上测试，且邻区分辨任务为二元分类，不能代表更复杂的决策空间（如多类型、协商、记忆等）。
- **单个LLM代理**：仅注入一个LLM代理，不能反映多LLM代理交互引起的集体行为变化或放大效应。
- **LLM多样性有限**：仅测试了Qwen3.5系列，未涵盖其他系列或商用API（如GPT‑4），无法评估跨模型泛化能力。
- **未深入探索提示工程与工具设计**：系统提示词和工具接口固定，未进行灵敏度分析，提示敏感性可能影响结论稳健性。
- **计算开销阻碍实际应用**：7秒/次推理在几十步的小规模仿真中已导致巨大耗时，无法扩展到更大网格、更长时间或更多代理的仿真。
- **SMC分析浅尝辄止**：仅进行了瞬态分析，缺乏稳态分析、敏感性参数扫描或更复杂的性质查询。
- **实验统计力有限**：全仿真仅20次（由SMC自动满足宽度要求），可能无法检测微弱的动态差异；鲁棒性测试重复次数仅为3次，对随机故障的估计可能不充分。
