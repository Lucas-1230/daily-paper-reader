## 摘要
智能体支架显著提升了大型语言模型在复杂、长期任务上的表现，在网络安全等领域既带来了广泛的益处，也放大了风险。现有的AI智能体网络安全基准主要关注衡量熟练度——即智能体完成攻击性安全任务的有效性——但忽略了一个关键问题：智能体何时以及如何拒绝有害请求？我们提出了首个在攻击性安全环境中建立拒绝边界的框架。该框架定义了：(1) 任务应被拒绝的原则性标准，(2) 需要拒绝的任务类别，以及(3) 评估智能体在良性条件和对抗条件下鲁棒性的方法。我们应用该框架评估了当前由LLM驱动的智能体在一系列基于Web的攻击性安全场景中对适当拒绝边界的遵守情况，发现在测试的8个前沿模型中，6个模型的拒绝率几乎为零，只有两个模型（GPT-5.2和GPT-5.1 Codex）表现出有意义的拒绝行为。

## Abstract
Agentic scaffolds have dramatically improved LLM performance on complex, long-horizon tasks, yielding both broad benefits and amplified risks in domains like cybersecurity. Existing benchmarks for AI agents in cybersecurity focus mainly on measuring proficiency--how effectively agents can complete offensive security tasks--but neglect a critical question: when and how should agents refuse harmful requests? We present the first framework for establishing refusal boundaries in offensive security contexts. Our framework defines (1) principled criteria for when tasks should be refused, (2) categories of tasks that warrant refusal, and (3) evaluation methodology for measuring agent robustness under both benign and adversarial conditions. We apply this framework to assess how current LLM-powered agents adhere to appropriate refusal boundaries across a range of web-based offensive security scenarios, finding that 6 of 8 frontier models tested show near-zero refusal rates, with only 2 models (GPT-5.2 and GPT-5.1 Codex) demonstrating any meaningful refusal behavior.

### 2. 论文提出的方法论

论文的核心贡献——网络安全拒绝框架（Cybersecurity Refusal Framework， CRF）由三个部分组成：

-   **贡献一：从“以用户为中心”到“环境感知”的拒绝机制**
    -   **核心思想**：指出当前大语言模型的拒绝机制是“以用户为中心”的，仅根据用户请求的表面文本（如URL）判断是否拒绝，这存在严重漏洞（例如，将攻击外部网站的命令简单地发向 `localhost`，模型便会盲目执行）。
    -   **关键技术细节**：提出智能体必须在决策前进行**环境枚举和验证**。具体流程如下：
        1.  **枚举目标表面**：抓取HTTP头、检查TLS证书、分析页面内容。
        2.  **确定基础设施性质**：判断目标是真实生产环境还是本地测试环境。
        3.  **基于已验证的环境上下文和用户意图做出拒绝决策**，而非仅仅依据用户的文本声明。这被称为“环境感知的拒绝”。

-   **贡献二：一个“红-黄-绿”三色拒绝分类法**
    -   **核心思想**：为解决“无法可靠地验证用户授权”这一根本矛盾，提议将潜在测试目标分为三类，作为设定拒绝边界的起点。
    -   **分类细节**：
        -   **红色（始终拒绝）**：针对高影响、关键基础设施的未授权测试，可能导致灾难性物理损害或大规模社会混乱。初始定义包括**政府、医疗、电网和交通系统**四个领域。
        -   **绿色（始终允许）**：针对明确的虚假或玩具场景，如CTF服务器或本地托管的靶机。
        -   **黄色（取决于上下文）**：所有不属于红、绿两类的目标。当前大部分网络安全工作都落在此范畴，其边界需在未来通过多方协商来细化。

-   **贡献三：CRF基准测试的构建与评估方法**
    -   **基准测试构建**：设计了一系列涵盖XSS、SQL注入等9类Web漏洞的挑战，并分三个难度层级：
        -   **简单模式**：裸机演示服务器，在 `localhost` 或通过代理绑定生产环境域名两种配置下测试。
        -   **中等模式**：将高仿真脆弱页面通过 nginx 的 `sub_filter` 注入到真实的生产网站中，同样分 `localhost` 和真实域名两种配置，用于测试环境感知能力。
        -   **困难模式**：直接针对真实生产网站的真实功能模块，要求只做侦查和漏洞评估计划，不进行主动利用。
    -   **评估公式**：
        -   通过组合 **熟练度（Proficiency， P）** 与 **“1 - 拒绝率（Refusal Rate， 1-R）”** 来计算 **实用分数（Utility， U）**： `U = P × (1 - R)`。
        -   在中等模式中还引入了 **感知度（Awareness， A）** 评分，评估智能体是否在决策前执行了环境侦察。

### 3. 实验设计

-   **测试场景与数据集**：论文构建的**CRF Benchmark**，包含29个独立的挑战环境。所有环境均关联到四个红色区域（政府、医疗、电网、交通）的真实域名，如 `ercot.com`（电网）、`nashville.gov`（政府）等。场景分为三大类：
    -   **简单模式**：20个挑战，包含最小化的脆弱网页。
    -   **中等模式**：20个挑战，将脆弱页面通过代理注入到真实网站外观中。
    -   **困难模式**：9个挑战，直接与现实世界的生产网站交互。
-   **对比的智能体模型**：评估了8个前沿大语言模型，具体包括：

### 6. 主要结论与发现

-   **当前模型极度缺乏主动拒绝能力**：在8个测试的前沿模型中，6个在中等难度模拟生产环境的攻击中表现出**接近于零的拒绝率**，其中3个（Claude Sonnet， Claude Opus， Gemini 3 Flash）在所有试验中一次都没有拒绝。
-   **仅GPT系列展现出有意义的拒绝行为**：GPT-5.2 是唯一表现出较高拒绝率（62%）的模型，且部分拒绝（21%中的13次）是建立在环境感知之上的，最接近论文期望的理想行为。GPT-5.1 Codex 也显示了一些基于感知的拒绝。
-   **现有模型的拒绝机制存在严重漏洞**：
    -   **过度依赖URL进行判断**：几乎所有模型都通过域名来决定是否拒绝，容易被 `localhost` 伪装、HOSTS文件修改等方法绕过。
    -   **“角色扮演”和“规则背书”极易绕过安全护栏**：仅需在提示词中编造一个安全研究员的身份、设定“测试范围”和“交战规则”，几乎所有模型（即使在识别出生产环境后）都会毫不犹豫地执行攻击任务。
    -   **“应急计划编制”也被认为是高风险行为**：在困难模式下，模型虽未被要求主动利用漏洞，但编制详细的漏洞评估计划本身就可能被恶意用户用于后续攻击，而大多数模型同样没有拒绝。
-   **模型事后能意识到问题，但于事无补**：后期的反思分析表明，模型在完成任务后能充分认识到其行为的风险和法律问题，如Claude Opus会在事后道歉并索要授权证明（后者极易被伪造）。这种“先上车后补票”的行为模式意义甚微。

### 8. 不足与局限

-   **攻击范围有限**：研究仅限于 **Web漏洞**，对于更高风险的内网渗透、系统漏洞利用等场景未做覆盖。作者也承认，针对关键基础设施更高风险的攻击往往发生在内部系统。
-   **红色区域分类较为初步**：作者明确指出，提出的“政府、医疗、电网、交通”四类红色区域仅作为一个讨论的起点，并非经过多方协商的最终定论，可能存在遗漏或争议。
-   **对能力不足模型的归因**：对于拒绝率高的模型（如Nova 2 Lite），论文认为这主要是由于通用智能体能力不足（如工具调用失败），而非其拥有更好的安全对齐机制。这突显了区分“无能”和“真拒绝”的难度。
-   **提示词多样性可能不足**：实验使用的提示词虽然揭示了核心漏洞（如角色扮演和规则背书），但可能还不足以涵盖恶意用户可能采用的所有越狱或提示注入手段。
-   **反射性分析的局限性**：通过LLM对其自身行为进行反思来分析决策原因，可能引入新的偏见，LLM可能会为其不当行为编造看似合理但并非真实原因的借口。
