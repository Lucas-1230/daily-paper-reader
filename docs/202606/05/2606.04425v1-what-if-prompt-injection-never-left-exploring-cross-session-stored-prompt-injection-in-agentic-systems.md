## 摘要
现代智能体系统将大语言模型从局限于会话的助手转变为有状态的系统，这些系统通过记忆、文件系统、工具和其他长期存在的上下文制品，在会话之间维持并演化共享的世界状态。这一转变从根本上扩大了提示注入的攻击面。然而，先前关于提示注入的工作主要关注单个会话内的模型级威胁，忽视了跨会话持久化系统状态如何从根本上改变智能体系统的系统级风险。受网络系统中的存储型跨站脚本攻击启发，我们引入了跨会话存储型提示注入，即成功的注入可以持久存在于智能体系统状态中，并在原始攻击者交互结束后很久仍能静默地影响未来的执行。为系统研究这一威胁，我们形式化了存储型提示注入，并提出了一个关于对抗性内容如何在会话间持久存在并影响智能体系统的分类体系。我们还开发了一个基准测试和沙箱工具包，用于评估存储型提示注入的风险，从而能够对不同模型、攻击目标和持久化渠道下的攻击成功率进行定量分析。我们的发现强调，持久性将提示注入从一种短暂的模型级威胁转变为嵌入智能体执行状态的长期系统级漏洞。我们希望这项工作能引起人们对这一新兴威胁的更广泛关注，并激励社区系统研究并缓解由智能体系统中的持久性引发的系统风险。

## Abstract
Modern agentic systems transform LLMs from session-bounded assistants into stateful systems that persist and evolve shared world state across sessions through memories, filesystems, tools, and other long-lived contextual artifacts. This shift fundamentally expands the attack surface of prompt injection. However, prior works on prompt injection have largely focused on model-level threats within a single session, overlooking how cross-session persistent system state fundamentally changes the system-level risk of agentic systems. Inspired by stored cross-site scripting in web systems, we introduce cross-session stored prompt injection, where a successful injection can persist within agentic system state and silently influence future executions long after the original attacker interaction has ended. To systematically study this threat, we formalize stored prompt injection and develop a taxonomy of how adversarial content persists and affects agentic systems across sessions. We further develop a benchmark and sandbox toolkit to evaluate the risks of stored prompt injection, enabling quantitative analysis of attack success across different models, attack goals, and persistence channels. Our findings highlight that persistence transforms prompt injection from an ephemeral model-level threat into a long-lived system-level vulnerability embedded within agent execution state. We hope this work draws broader attention to this emerging threat and motivates the community to systematically study and mitigate system risks arising from persistence in agentic systems.

# 二、论文提出的方法论

- **核心思想**  
  存储型提示注入是一种状态性攻击：攻击者在一次交互中通过外部输入将恶意负载写入持久上下文，然后在后续正常会话中，当系统上下文构造函数自动或条件性地将该内容纳入模型上下文时，攻击被激活。
- **形式化定义**  
  - **持久上下文**：任何在系统内保存且可能在未来执行中被上下文构造函数 $A$ 纳入模型上下文的信息。
  - **存储型提示注入**：当攻击者控制的恶意内容 $p$ 被写入或修改持久上下文 $P$，并在后续执行中被纳入代理上下文 $x_t = A(q_t, \cdot, P)$，从而通过模型 $M$ 改变代理行为。
- **系统与威胁模型**  
  代理系统被建模为上下文构造管道。攻击者能力被抽象为从不受信区域向持久上下文通道的“写原语”：$w: u \mapsto p \in P$。攻击目标不是劫持单次调用，而是建立对未来代理行为的持久影响；攻击者无需特权访问模型或直接改写上下文，只需污染持久状态。
- **攻击分类法**  
  为系统刻画攻击生命周期，提出三维分类：
  1. **注入源**：用户提供的内容、外部消费的内容、供应链可安装工具上下文。
  2. **持久上下文通道**：代理记忆（工作记忆 vs. 档案记忆）、工具可见状态（工具描述、工具响应等）、文件型上下文（如 `AGENTS.md` 等约定文件或工作区文件）。
  3. **上下文纳入机制**：直接加载（无条件纳入，构成强持久化高风险面）与条件加载（依赖检索或工具调用才纳入，攻击具有随机性）。
  4. **攻击危害类别**：事实操纵、偏好操纵、行动范围操纵。
- **生命周期分解**  
  攻击被分解为**写入、持久化、纳入、激活**四个阶段，为后续定量评估奠定理论基础。

# 三、实验设计

- **数据集与场景**  
  本文手工设计基准测试 **SPI-Benchmark**，涵盖 3 个应用场景：
  - 电子商务
  - 旅行预订
  - 金融投资组合管理  
  共构造 **162 个独特攻击案例**，每个案例均预定义标准答案/行为、攻击声明和受害者任务。
- **基准测试结构**  
  采用四阶段评估流水线：
  1. **环境搭建**：为各场景初始化沙箱环境和工具。
  2. **注入会话**：提交攻击查询，观察恶意内容是否被写入目标持久通道。
  3. **会话重置**：清空对话历史，但保留持久存储，启动新会话。
  4. **激活会话**：提交干净的受害者查询，记录代理响应和工具调用，判断存储内容是否影响行为。
- **对比方法**  
  测试了三款代表性大语言模型：**GLM-5.1**、**GPT-5-mini** 和 **MiniMax-M2.7**，并系统对照以下因素：
  - 攻击目标（事实操纵、偏好操纵、行动范围操纵）
  - 持久通道（工作记忆－直接加载、档案记忆－条件加载、文件型上下文－直接加载）
  - 攻击类型（朴素注入 vs. 上下文伪装）
  - 应用场景（电子商务、旅行、金融）
- **评价指标**  
  依据生命周期定义：
  - 写入成功率 **(WSR)** = $N_{\text{written}} / N_{\text{attempts}}$
  - 纳入率 **(IR)** = $N_{\text{incorporated}} / N_{\text{written}}$
  - 激活率 **(AR)** = $N_{\text{activated}} / N_{\text{incorporated}}$
  - 端到端攻击成功率 **(E2E-ASR)** = $\text{WSR} \times \text{IR} \times \text{AR}$

# 六、论文的主要结论与发现

- **整体脆弱性**：三个模型的端到端攻击成功率在 32.1%–42.0% 之间，证实存储型提示注入具有真实且跨模型的可行性。
- **生命周期瓶颈差异**：不同模型的薄弱环节不同。如 GLM-5.1 写入率较低但纳入和激活率高，而 MiniMax-M2.7 写入率最高却因纳入瓶颈导致整体成功率相近，说明写入与激活阶段高度解耦。
- **攻击目标影响**：事实操纵最为高效（AR 达 100%），偏好操纵极难成功（AR 0–40%），行动范围操纵存在较大模型间差异。
- **持久通道决定风险等级**：直接加载型通道（工作记忆、文件型上下文）的攻击成功率一致高于条件加载型通道（档案记忆），表明“强持久化且无条件纳入”的系统状态是主要风险面。
- **攻击伪装有效提升写入**：上下文伪装相比朴素注入能显著提高 WSR，但对后续纳入与激活的提升有限，揭示写入阶段是当前的主要屏障。
- **场景鲁棒性**：多数模型在不同应用场景下表现出较一致的脆弱性，仅在金融场景中 GPT-5-mini 安全性有所提高。
