## 摘要
本文探讨了基于智能体的AI工具在网络安全领域的价值。我们评估了一个基于通用生成式AI大语言模型（GenAI）的智能体在由三种不同的Ollama托管的通用开源模型驱动时的有效性。我们使用精确率、召回率、误报数以及基于这些指标相互作用计算出的综合得分，将每个智能体的性能与现有经过验证的静态应用安全测试（SAST）工具Bandit的基线性能进行对比。我们的研究结果反驳了这样一种观点：即现代开源GenAI LLM驱动的智能体目前适用于在现实条件下执行SAST扫描这一专业任务。

## Abstract
This paper explores the value of agentic AI tools for cybersecurity purposes. We evaluate the efficacy of a general-purpose GenAI Large Language Model- (GenAI-) based agent when powered by three different Ollama-hosted general-purpose open source models. We assess each agent's performance using precision, recall, false positive count, and a calculated composite score based upon the interplay of the captured metrics, against the baseline performance of an existing, vetted Static Application Security Testing (SAST) tool, Bandit. Our findings refute the notion that a modern open-source GenAI LLM-based agent is currently suitable for the specialized task of SAST scanning under realistic conditions.

*   **核心算法与评估公式**：
    *   **匹配规则**：如果模型检测出一个Bandit标记为“中/高”严重度的漏洞，计为完全匹配（1分）或部分匹配（0.5分）。
    *   **召回率**：$ \text{Recall} = \frac{\text{模型检测出的（中/高）漏洞数}}{\text{Bandit发现的（中/高）漏洞总数}} $
    *   **误报率**：$ \text{FP Rate} = \frac{\text{人工判定的误报数}}{\text{模型发现的全部潜在问题数}} $
    *   **综合得分**：$ \text{Composite Score} = \text{Recall} - \text{FP Rate} $，得分越高，性能越好。

### 6. 论文的主要结论与发现
*   **核心发现：当前开源通用LLM智能体无法取代SAST工具**。在所有评估指标上，大模型智能体的表现均显著逊于传统工具`Bandit`。

*   **最终排名**：在三个模型中，`qwen2.5` 综合表现相对最优，但即便如此，其性能也远未达到可商用的标准。

*   **实验设计严谨**：采用固定提示词和文件，与成熟基线工具进行同等条件下的头对头比较，评估指标（召回、误报、综合得分）清晰且可量化。

### 8. 不足与局限
*   **提示词策略单一**：仅使用了一套固定的、未经优化的“安全专家”提示词，因此实验只能证明该固定策略的无效性，未能探索提示工程能否大幅提升性能。
*   **比对基线单一**：仅使用了`Bandit`作为唯一参照点，其不一定是绝对完美的“真值”，可能遗漏一些漏洞类型。
*   **模型与任务范围局限性**：实验仅限于三个开源模型和SAST任务，未涵盖闭源模型（如GPT-4、Claude）或动态测试（DAST）等其他安全任务。

*   **智能体设计过于“朴素”**：作者承认，该智能体设计可能过于简单，无法代表更复杂智能体系统（如多智能体协作或融合外部知识的智能体）的性能上限。
