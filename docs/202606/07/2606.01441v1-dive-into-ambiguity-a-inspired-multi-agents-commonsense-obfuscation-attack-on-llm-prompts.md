## 摘要
大型语言模型 (LLM) 在推理和知识密集型任务中表现出色，但仍易受保持意图但引发常识幻觉的提示级对抗攻击。这一漏洞亟待解决，因为 LLM 正迅速融入安全性至关重要的领域，在这些领域中，事实可靠性不容妥协。现有攻击方法要么效率低下，要么无法捕捉现实世界对手的自适应策略。我们提出了受 A* 启发的事实错误诱导框架，这是一个生成语义对齐但混淆提示的框架。其核心是一种分层重写策略，由动态语义分散系数 $γ$ 引导，该系数在早期保守编辑与后期激进混淆之间取得平衡，遵循反向模拟退火计划。为了增强可解释性，我们进一步引入了智能体机制标注，它发现并提炼对抗机制，提供可解释的逆向优化。理论上，我们证明了提示重写遵循收缩递归，随着 $γ$ 减小，导致语义崩溃。实证上，在各种 LLM 中，我们的方法在所需尝试次数更少的情况下实现了比穷尽探索更高的攻击成功率，展示了效率与有效性。

## Abstract
Large language models (LLMs) excel in reasoning and knowledge-intensive tasks but remain vulnerable to prompt-level adversarial attacks that preserve intent while triggering commonsense hallucinations. This vulnerability is urgent, as LLMs are rapidly integrated into safety-critical domains where factual reliability is non-negotiable. Existing attack methods either lack efficiency or fail to capture the adaptive strategies of real-world adversaries. We propose an A*-inspired Factual Error Induction Framework, a framework for generating semantically aligned yet obfuscated prompts. At its core is a Hierarchical Rewrite Strategy guided by a dynamic semantic dispersion coefficient $γ$ that balances conservative edits early with aggressive obfuscations later, following a reverse simulated annealing schedule. To enhance interpretability, we further introduce Agentic Mechanism Labeling, which discovers and refines adversarial mechanisms, offering interpretable reverse optimization. Theoretically, we prove that prompt rewriting follows a contractive recurrence, leading to semantic collapse as $γ$ decreases. Empirically, across diverse LLMs, our method achieves higher attack success rates than exhaustive exploration while requiring fewer attempts, demonstrating both efficiency and effectiveness.

- **背景**：大型语言模型（LLM）广泛应用于信息检索、决策支持等安全关键领域，但容易在提示扰动下产生“常识幻觉”，即对简单常识问题给出错误答案，影响可靠性和可信度。
- **问题现状**：现有对抗提示攻击方法（如随机替换、穷举搜索）效率低、缺乏精细控制，且难以模拟真实攻击者的自适应策略。尤其在黑盒商业模型下（如 GPT-4o、Claude），无法获取模型权重，只能基于 API 进行攻击。
- **核心目标**：设计一种高效、可解释的提示级对抗攻击框架，在保留原意的前提下，通过逐步增加语义混淆来诱导 LLM 产生常识错误，同时尽可能减少 API 调用次数。

## 2. 方法论
### 2.1 整体框架：A* 启发的事实错误诱导框架
将对抗提示生成视为在语义空间中的离散搜索问题，从原始提示 $x_0$ 出发，逐步重写候选提示，目标是找到使目标模型 $F$ 输出与正确答案 $y^*$ 不一致的对抗提示 $\tilde{x}$，且需限制尝试次数 $N$。

**搜索核心**：
- 每个节点代表一个候选提示，边为语义保留的编辑操作（同义词替换、释义、插入干扰修饰语等）。
- 使用优先队列维护候选节点，按评估函数 $f(n) = g(n) + h(n)$ 排序（$g(n)$ 为累积编辑代价，$h(n)$ 为启发式距离估计）。
- 启发式距离由多个归一化组件加权组合构成：编辑距离、输入语义质心距离、输出语义距离等，实现了多种代价函数变体（如 edit-only、centroid-only、out-only、combined）。

### 2.2 语义分散系数与动态重写攻击性调度
定义语义分散系数 $\gamma_t$：

$$\gamma_t = \frac{1}{N}\sum_{i=1}^N \| \phi(x_t^{(i)}) - \bar{v}_t \|^2$$

其中 $\bar{v}_t$ 为候选提示嵌入的局部质心。该系数衡量当前搜索池中语义变体的多样性。

**理论证明**（定理1及其推论）：在假设（语义收缩性、变体继承、嵌入连续性）下，$\gamma_t$ 满足 $\gamma_{t+1} \le L\gamma_t + K\delta$，其中 $L\in[0,1)$。随着搜索进行，$\gamma_t$ 逐渐下降并收敛至一个有界区域，即发生“语义崩溃”。这要求在后期阶段增强重写多样性。

**分层重写策略**：
- **Level 1（保守）**：单代理轻度改写（同义词、语序调整），保持高语义保真。
- **Level 2（适中）**：双代理产生不同变体，再由评判代理合成，注入受控的歧义。
- **Level 3（激进）**：多代理辩论（MAD），多代理生成多种混淆变体并融合，最大化混淆。

**动态选择机制**：根据当前 $\gamma_t$ 值，使用调度器（线性、softmax、带热身的二次调度器）决定各等级被选中的概率。 $\gamma_t$ 高时倾向于保守（Level 1），低时倾向于激进（Level 3）。这类似于逆向模拟退火，随搜索进展逐渐增加重写攻击性。

**多代理辩论（MAD）的理论保证**：证明在 MAD 合成下，错误概率单调不减，即 $Pr(I_{t+1}=1) \ge Pr(I_t=1)$，且当引入非冗余错误触发特征时严格提升，因此有助于对抗语义崩溃。

### 2.3 智能体机制标注（AML）框架
目标：自动发现、标注和提炼对抗提示中导致模型出错的机制，并将这些机制反馈到重写过程以进行逆向优化。

**工作流程**：
1. **分类法探索**：从一批对抗提示中，LLM 在两种模式下提出候选机制 —— 可量化标注（关注可计算的识别特征）和抽象标注（关注概念性策略，如过度阐释、修辞框架等）。
2. **机制分类法挖掘**：统计各机制出现概率，丢弃低频项，合并重叠机制。
3. **提示编辑**：如尚未达到停止标准，针对覆盖不足的机制生成新的编辑提示。
4. **收敛检查**：当覆盖增益、分类法变化、标注者间一致性满足阈值时停止。

最终产出精确的分类法和机制的概率分布，可用于指导后续攻击。

## 6. 主要结论与发现
- **代价函数选择**：闭源模型（GPT 系列）对 out-only 代价函数响应最佳，成功率高；开源模型则在 combined（多信号融合）下表现更好。随机搜索在各模型上均表现最差，证明启发式引导的有效性。
- **$\gamma$ 调度策略**：动态调度显著优于固定激进策略。闭源模型在二次调度下最优，开源模型在 softmax 调度下最优，说明自适应攻击性控制关键。
- **语义崩溃验证**：实验观察到 $\gamma_t$ 随搜索进行而下降，确认了理论推导的语义收缩现象；无调度时收敛更慢且不稳定。
- **AML 有效性**：抽象标注的覆盖度和复用率随轮次增加而提升；可量化标注在保证高覆盖的同时，准确率也稳步提升。单机制引导注入大幅提升成功率（相对提升 114.8%），而多机制则因信号重叠收益下降。
- **效率与效果**：所提方法在更少 LLM 调用次数（例如使用调度）下达到甚至超过固定激进策略的成功率，尤其对闭源模型优势明显。

## 7. 优点
- **创新启发式融合**：将 A* 搜索思想与语义空间中的编辑操作结合，设计出多种可组合的代价函数，克服传统随机搜索和穷举的低效。
- **理论与实践结合**：定理证明语义收缩现象，并据此设计动态调度策略，使攻击具有理论指导下的自适应性。
- **多代理协同**：构建三层次改写和 MAD 机制，兼顾语义保持与混淆强度，对抗语义崩溃。
- **可解释性机制**：AML 框架能自动挖掘并量化对抗机制，不仅提升攻击成功率，还为防御提供见解。
- **实验详尽**：在不同模型、不同代价函数、不同调度策略上进行了系统对比，并有消融实验和超参数分析。

## 8. 不足与局限
- **数据集规模小**：只使用了 128 个手工对，泛化性有待更大规模、更多元常识测试集的验证。
- **模型覆盖有限**：闭源模型只测试了 GPT 系列，未包括 Claude、Gemini 等主流商业模型；开源模型也只测试了 7B 和 13B，缺乏更大规模模型的检验。
- **攻击类型单一**：专注于常识诱导错误，未涉及有害内容生成、越狱等其他对抗场景。
- **AML 依赖 LLM**：机制标注本身依赖 LLM 的判断，存在自评偏差和幻觉风险；抽象机制的正确性验证不足。
- **黑盒假设边界**：虽然声称黑盒，但重写代理本身也是 LLM，可能需要额外的 API 成本；未考虑更严格的防御措施（如系统指令过滤）。
- **理论假设较强**：语义收缩性假设（H1）在实践中可能不完全成立，理论的普适性还可探讨。
