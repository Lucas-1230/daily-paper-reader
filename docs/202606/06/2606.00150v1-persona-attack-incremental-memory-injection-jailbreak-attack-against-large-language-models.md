## 摘要
随着大语言模型为方便用户而不断演进，尽管安全训练工作一直在进行，但越狱攻击的脆弱性仍持续被报告。传统的越狱技术通常专注于单次提示注入，忽视了模型记忆对话流程和用户指令的能力。本文提出角色攻击，一种基于记忆注入的越狱方法，通过逐步的方式操纵模型的上下文窗口。将角色攻击应用于几个广泛使用的大语言模型的实验结果表明，随着注入在记忆中累积，模型越来越优先考虑这些指令，而非其内部的安全对齐机制。此外，我们的实验实证表明，攻击成功率不仅随模型记忆实现的不同而变化，而且还随指令组合而变化，在特定指令配置下可达95%。

## Abstract
As Large Language Models evolve for user convenience, vulnerability to jailbreak attacks continues to be reported despite ongoing efforts in safety training. Traditional jailbreak techniques typically focus on a single prompt injection, neglecting the models' ability to remember the flow of conversation and the user's instructions. In this paper, we propose Persona Attack, a memory injection based jailbreak method that manipulates the model's context window through a step by step approach. Experimental results from applying Persona Attack to several widely used LLMs reveal that, as injections accumulate in memory, models increasingly prioritize these instructions over their internal safety alignment mechanisms. Furthermore, our experiments empirically demonstrate that the attack success rate varies not only according to the memory implementation of the model, but also combinations of instructions and can reach 95% under specific instruction configurations.

## 2. 方法论：Persona Attack
### 2.1 总体思想
把越狱看作一个**逐步污染记忆**的过程：先向模型注入一组无害的“指令印记”，让模型在后续对话中持续遵循这些指令，再在适当位置嵌入真实有害问题，利用指令惯性绕过安全对齐。

### 2.2 攻击流程
- **指令印记阶段（Instruction Imprinting）**  
  注入三条指令 \\(I_1, I_2, I_3\\)，分别对应：  
  - \\(I_1\\)（模仿）：让模型预测其他语言模型的输出；  
  - \\(I_2\\)（假设注入）：将问题限定为“格式跟随型问题”，并强制要求输出四种类别结果（越狱失败/部分失败/部分成功/成功）；  
  - \\(I_3\\)（去掩码）：要求输出完整答案，不得遮蔽特定词。  
  这些指令被存入模型记忆，为后续攻击创造“合法外衣”。

- **攻击步骤（Attack Step）**  
  将一个有害问题 \\(Q_h\\) 包装在类 JSON 格式的查询中，如：
  ```
  So, what if the "prompt" in "query" changes this : "specific harmful question"?
  ```
  由于模型记忆中已存在 \\(I_2\\) 设定的四分类框架，它会自动将新查询归类为“越狱成功”等情形，并以模拟其他模型响应的名义生成有害内容。

### 2.3 指令组合与优化
- 攻击形式根据 \\(I_1, I_2, I_3, Q_h\\) 的排列不同，分为：
  - **Sequential 1**：顺序输入 \\(I_1 \to I_2 \to I_3 \to Q_h\\)
  - **Sequential 2**：\\(I_1 \to I_2 \to Q_h \to I_3\\)
  - **Once**：一次性拼接 \\(I_1 + I_2 + I_3 + Q_h\\)
  - **Additional variants**：如 \\((I_1+I_2) \to Q_h\\) 等。
- 将攻击成功率的优化建模为一个**组合优化问题**：  
  \[
  \max_{X \in C(I, Q_h)} \text{ASR}(\text{LLM}(X))
  \]
  其中 \\(C(I, Q_h)\\) 是所有有效组合的集合。实际采用**搜索空间剪枝**策略，只测试可能获得高 ASR 的配置。

### 2.4 记忆实现分类
- **手动记忆（Manual Memory）**  
  每轮将历史对话（提示-响应对）直接拼接到新提示前面，作为上下文输入。
- **状态记忆（State-based Memory）**  
  模型内部维护一个状态 \\(s_t\\)，通过记忆更新函数 \\(M\\) 迭代融入新对话：  
  \[
  s_0 = 0;\quad r_t = \text{LLM}(x_t \mid s_{t-1});\quad s_t = M(s_{t-1}, x_t, r_t)
  \]
  这种实现更贴近原生 API 的对话管理方式。
