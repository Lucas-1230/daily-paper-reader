## 摘要
代理技能使基于大语言模型的编程代理能够从第三方包中获取特定领域的能力，但也引入了新的供应链攻击面。我们提出了 PhantomSkill 攻击框架，该框架将恶意行为隐藏在技能的辅助资源中，而非其文本描述中。其核心技术 VulMask 将明显的恶意脚本重写为看似存在漏洞的实现，其恶意行为仅在攻击者控制的触发条件下激活。这种设计将可见信号从明确的恶意意图转变为看似普通的非安全代码。在代表性主机技能、攻击目标、编程代理、生成模型和自动审查者的测试中，VulMask 在保持良性效用的同时，相比明显的恶意脚本，降低了警告和恶意软件级别的检测率。我们的结果表明，技能生态系统需要资源级别的审查、执行时隔离，并将代理技能中可被利用的漏洞视为潜在的恶意载荷的安全策略。

## Abstract
Agent skills allow LLM-based coding agents to acquire domain-specific capabilities from third-party packages, but they also introduce a new supply-chain attack surface. We present PhantomSkill, an attack framework that hides malicious behavior in a skill's auxiliary resources rather than in its textual description. Its core technique, VulMask, rewrites overt malicious scripts into vulnerability-shaped implementations whose malicious behavior is activated only under attacker-controlled trigger conditions. This design shifts the visible signal from explicit malicious intent to ordinary-looking insecure code. Across representative host skills, attack goals, coding agents, generation models, and automated reviewers, VulMask preserves benign utility while reducing warning and malware-level detection compared with overt malicious scripts. Our results show that skill ecosystems require resource-level vetting, execution-time containment, and security policies that treat exploitable vulnerabilities in agent skills as potential malicious payloads.

### 1. 论文的核心问题与整体含义

*   **研究背景**：大语言模型（LLM）被越来越多地用作编程代理的核心，例如 Claude Code、Cursor 等。为了扩展代理的能力，Anthropic 引入了“代理技能”包格式，允许代理从第三方获取可复用的领域知识，这些技能包除了包含说明文件（`SKILL.md`）外，常常还有可执行的脚本（辅助资源）。
*   **核心问题**：这种便捷性引入了一个新的供应链攻击面。现有的攻击研究多集中于在文本描述中植入恶意指令（提示词注入），或使用明显的恶意脚本。然而，现代编程代理或审查者可能在执行或安装前检查这些脚本，使得此类显式的恶意行为容易被发现。
*   **整体含义**：本研究旨在探索一种更隐蔽的攻击方式，即攻击者不直接使用明显的恶意代码，而是将恶意行为伪装成看似普通的软件漏洞，从而绕过人为检查和自动审查系统，揭示当前代理技能生态系统在资源层面的安全风险。

### 2. 论文提出的方法论

论文的核心贡献是提出了 **PhantomSkill** 攻击框架及其关键技术 **VulMask**。

*   **核心思想**：VulMask 的核心理念是进行“信号转移”——将代码审查者的注意力从“明确的恶意意图”转移到“普通的非安全编码”上。审查者对明目张胆的恶意软件非常敏感，但对存在漏洞（如命令注入、信息泄露）的代码往往只会标记为不安全，而不是恶意攻击。
*   **关键技术细节（VulMask流程）**：
    1.  **漏洞类别选择**：攻击者首先选定与预期恶意行为相关的一组漏洞类别 `W`（如 `CWE-522`：凭证泄露，`CWE-94`：命令注入）。
    2.  **载荷重写**：一个由LLM驱动的生成器 `g` 将攻击者原本的、明显的恶意脚本 `p*` 重写为一个“漏洞形态”的载荷 `pm`。公式表示为：`pm = g(p*, W)`。`pm` 保留了攻击者的恶意副作用，但通过一个可被攻击者控制的触发条件来激活，而非使用显式的恶意逻辑。
    3.  **嵌入与整合**：VulMask 并非用恶意载荷完全替换原有脚本，而是将 `pm` 嵌入到主机技能的原始资源 `rh` 中，形成被篡改的资源 `rm`。公式表示为：`rm = Embed(rh, pm)`。此举旨在保留技能的名称、路径和正常的输入输出行为，使其在被触发前仍能正常执行原本的良性功能。
    4.  **验证**：最终生成的恶意技能 `sVulMask` 会在一个隔离的测试环境中进行验证，以确保在无触发条件下技能功能正常（效用保留），而在攻击者设定的触发条件 `θ` 下，恶意行为能够被成功激活。

### 6. 论文的主要结论与发现

*   **有效性**：VulMask 成功地展示了高度隐蔽的攻击能力。它在保持高攻击成功率的同时，显著降低了编程代理的警告率和恶意软件的检出率，优于提示词注入和明显的恶意脚本。
*   **隐蔽性原理成立**：实验证明了其核心假设，即自动审查者倾向于将经VulMask处理后的载荷标记为“漏洞”而非“恶意软件”，实现了从恶意意图到普通不安全代码的信号转移。
*   **泛化能力**：VulMask 的攻击效果在不同攻击目标、不同载荷生成模型上均能有效迁移和泛化，不是针对单一场景的特例。
*   **防御现状**：当前防御措施存在盲区。对`SKILL.md`的扫描不足以抵御此类攻击。部分强大的代理（如基于Opus-4.7的Cursor）和安全扫描器（如Snyk）仍能有效检测，但多数代理和审查者存在明显弱点。
*   **安全启示**：技能生态系统的安全不能仅依赖文本审查。辅助资源应被视为一级攻击面，在代理环境中，一个看似可被利用的漏洞在功能上等同于一个恶意载荷。

### 8. 不足与局限

*   **实验覆盖范围**：评估主要针对代表性编程代理和生态系统，未涵盖所有新兴的代理平台和安全工具，未来不同的执行模型或权限系统可能影响攻击效果。
