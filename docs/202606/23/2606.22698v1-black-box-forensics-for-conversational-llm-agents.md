## 摘要
随着大语言模型驱动的欺诈活动日益猖獗，对话式大语言模型代理的黑盒取证为隐藏在匿名端点背后的系统提供了一条追责途径。识别聊天机器人端点背后的基础模型（归因），无需访问模型参数或知晓隐藏的系统提示，将使调查人员能够将 AI 驱动的诈骗追溯到提供模型支持的供应商。检测两个端点何时运行完全相同的系统提示（指纹识别），即使是新颖未见过的提示，也能将单个诈骗关联到犯罪网络，并揭露隐蔽的 API 变更。我们对这两种能力进行了实证研究。我们的归因分类器通过几轮非对抗性对话，就能以 98% 的准确率识别出代理背后的基础模型。系统提示的归因虽然可能，但需要针对每个提示重新训练大量数据；野外的系统提示是无边界且不断变化的，使得这种方法成本高昂。为了应对这种更开放的环境，我们的交叉编码器指纹识别方法在完全未见过的系统提示上实现了 0.768 的 AUC 和 0.703 的 F1 值，而从每个目标代理聚合 50 次交互对话将 AUC 提升至 0.943。因此，具有未见系统提示的对话代理可以通过几轮普通对话以稳健的准确度进行指纹识别。

## Abstract
As LLM-powered scams proliferate, black-box forensics for conversational LLM agents offers a path to accountability for systems hidden behind anonymous endpoints. Identifying the base model behind a chatbot endpoint (attribution), without model parameter access or knowledge of the hidden system prompt, would let investigators trace AI-enabled scams back to the providers whose models power them. Detecting when two endpoints run the exact same system prompt (fingerprinting), even one novel and unseen, would link individual scams into criminal networks and expose silent API changes. We conduct an empirical investigation of both capabilities. Our attribution classifiers identify the base model behind an agent with 98% accuracy from a few turns of non-adversarial conversation. Attribution of system prompts, while possible, requires retraining on a large amount of data for each prompt; system prompts in the wild are unbounded and ever-changing, making this approach costly. To tackle this more open-ended setting, our cross-encoder fingerprinting method achieves an AUC of 0.768 and an F1 of 0.703 on entirely unseen system prompts, and aggregating 50 interaction conversations from each target agent boosts AUC to 0.943. Conversational agents with unseen system prompts can thus be fingerprinted with robust accuracy from a few turns of ordinary conversation.

- **核心问题**：随着大语言模型（LLM）驱动的会话诈骗大量涌现，执法与平台治理亟需**对匿名聊天端点进行“黑盒取证”**，即在不访问模型参数、未知系统提示、仅通过普通对话的情况下，追溯幕后模型和提示配置。
- **研究目标**：提出并系统评估两类互补能力：
  - **归因**：从**已知封闭候选集**中识别端点背后的基础模型或系统提示。
  - **指纹识别**：判断**两个端点是否运行完全相同的基础模型和系统提示**，即使这些提示是**训练阶段从未见过**的。
- **应用场景**：溯源诈骗网络、发现隐秘的 API 后端变更、辅助针对特定模型–提示组合的红队测试、关联不同平台上的诈骗活动。

### 2. 论文提出的方法论（核心思想与技术细节）

#### 2.1 系统模型与对话范式
- 定义对话代理为 `(模型 m, 系统提示 p)` 的组合。
- 采用**主动引导范式**：一个固定“侦探代理”（Qwen-4B-Instruct）以**非对抗、自然对话**（如客服、协商场景）与目标代理交互，控制话题并收集对话记录，以保证隐蔽性。

#### 2.2 归因方法
- **特征 + 线性分类器**：提取目标代理发言的 Unigram/字符级 TF‑IDF、29 项**笔测特征**（标点频率、句子长度分布、词汇丰富度等），用 **Ridge 逻辑回归**做分类。
- **微调 LLM 分类器**：在 Qwen-4B-Instruct 上添加 **LoRA** 适配器（秩 64）作为密集分类器。
- 分别评估**多分类**（模型族/多提示）和**二分类**（区分特定提示对）表现。

#### 2.3 指纹识别方法
- **基线方法**：
  - 基于**分布检验**（MMD Hamming 统计）的模型相等性测试。
  - n-gram 重叠阈值分类。
  - 基于 n-gram 相似度与笔测差异的传统机器学习（逻辑回归、GBDT、随机森林）。
- **双编码器**：用 MPNet / BERT 通过对比学习拉近相同代理对话的嵌入，拉远不同代理对话的嵌入，再通过阈值进行二分类。
- **交叉编码器**（核心方法）：将两段对话拼接输入 **ELECTRA-large**（或 BERT-base），输出相同/不同的对数概率，使用交叉熵损失微调。ELECTRA 的替换令牌检测预训练任务与指纹识别更匹配。截断为前 3 轮对话以适应 512 令牌窗口。
- **聚合推断**：对每个目标代理收集的 `k` 个对话对进行**多数投票**，随着 `k` 增加性能显著提升。

### 6. 主要结论与发现

- **归因能力**：
  - 基础模型归因准确率达 **98%**，即使同家族不同规模模型（GPT-OSS-20B/120B）也能以 95% 准确率区分。

  - 聚合 **50 次对话对**后 AUC 升至 **0.943**，展示强大的证据积累特性。
  - 方法对**话题迁移、侦探模型变更、采样参数变化、标点去除**较为鲁棒（AUC 下降 <0.03）。
  - 对**对话长度缩短**（尤其是 1-2 轮）和**改写攻击**（AUC 下降 ~0.10）敏感，但多数投票仍可在 10 对对话下恢复 AUC 至 0.94。

### 7. 优点（方法或实验设计上的亮点）

- **完全的隐蔽性与被动性**：不依赖越狱提示、对抗字符串或日志探测，仅用自然对话，避免被人类监控发现。
- **开放世界的指纹识别**：系统提示可以**零样本**识别，无需针对新配置重训，真正适应野外提示的无限变化。
- **主动引导范式**：由侦探代理主动控制话题，有效分离语义噪声与提示/模型的行为特征，增强了信号的纯净度。
- **全面的可控评估**：通过合成大规模、多维度的对话数据，精确控制 `(模型, 提示, 话题)` 变量，避免真实数据中无法观测混杂因子的风险。
- **强鲁棒性验证**：针对话题、侦探模型、采样参数、改写攻击等多重分布偏移进行了专项测试，提供了实用的性能边界。
- **可操作性强的聚合策略**：给出不同 `k` 值下的 ROC 曲线与决策阈值，可直接指导调查人员权衡交互次数与误报率。

### 8. 不足与局限

- **领域定制的合成数据**：实验全部基于合成的客服与协商对话，系统提示也由 GPT 生成；与真实诈骗场景中的高度特异性、反侦察提示或有差异，性能可能构成下界，但**真实性差距仍是一个不确定性**。
- **改写攻击敏感**：代理提示改写（proxy prompting）可使 AUC 下降约 0.10，说明方法对词级风格扰动仍有一定脆弱性；虽然聚合可恢复，但需更多交互。
- **上下文窗口限制**：使用 ELECTRA-large 的 512 令牌限制迫使截断对话，导致长对话后期性能下降；文中虽尝试了 Longformer 但训练不稳定，未能充分利用完整对话历史。

- **未在真实诈骗端点上测试**：尽管理由充分（法律、伦理限制），直接从论文无法了解在真实“野生的”诈骗代理上的性能。
