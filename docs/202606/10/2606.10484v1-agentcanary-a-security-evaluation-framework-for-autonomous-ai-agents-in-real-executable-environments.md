## 摘要
自主AI智能体推动了从对话到任务执行的转变，安全故障也从文本欺骗转向系统入侵。尽管安全评估对主动风险预防至关重要，但先前工作受到根本性瓶颈的制约，包括风险覆盖碎片化、静态或低保真度的执行环境，以及单一维度和粗粒度的评估指标。为应对这些挑战，我们提出AgentCanary——一个面向自主AI智能体的综合性安全评估框架。AgentCanary围绕三项贡献提供系统化解决方案。第一，全面的风险覆盖：我们引入一个正交的“入口×影响”风险分类法，将对抗性影响如何进入智能体与其最终造成的损害解耦，并将其实例化为与真实部署工作流对齐的场景任务集。第二，高保真度的真实可执行环境：智能体并非应对静态问答或模拟的工具响应，而是在动态配置的任务工件中与真实工具交互，跨越多个步骤的持久状态自然支持长周期攻击评估。第三，基于轨迹的多维度评估：评估依赖于完整的智能体轨迹而非回复文本或单次工具调用，能够沿三个正交维度——结果安全性、安全意识和任务效用——进行分解评分。我们在AgentCanary上评估了多种前沿模型，涵盖三个智能体框架，并采用多种既定对抗攻击方法。结果表明，当前智能体往往无法识别所面临的攻击，尤其在技能受损、持久状态和长周期执行攻击下表现明显，这为开发更可靠和安全的智能体系统提供了系统性基线。

## Abstract
Autonomous AI agents have driven the transition from conversation to task execution, shifting security failures from textual deception to system compromise. Although security evaluation is crucial for proactive risk prevention, prior work is constrained by fundamental bottlenecks, including fragmented risk coverage, static or low-fidelity execution environments, and single-dimensional and coarse-grained assessment metrics. To address these challenges, we propose AgentCanary, a comprehensive security evaluation framework for autonomous AI agents. AgentCanary provides a systematic solution along three contributions. First, comprehensive risk coverage: we introduce an orthogonal Entry $\times$ Impact risk taxonomy that decouples how adversarial influence enters the agent from what harm it ultimately causes, and instantiate it as a scenario-aligned task suite spanning realistic deployment workflows. Second, a high-fidelity real executable environment: rather than static Q&A or mocked tool responses, agents interact with real tools against dynamically provisioned task artifacts, with persistent state across multi-step interactions that naturally supports long-horizon attack evaluation. Third, trajectory-grounded multi-dimensional evaluation: evaluation consumes the full agent trajectory rather than the reply text or a single tool call, enabling decomposed scoring along three orthogonal dimensions, Outcome Safety, Security Awareness, and Task Utility. We evaluate a broad set of frontier models on AgentCanary against multiple established adversarial attack methods across three agent frameworks. The results reveal that current agents often fail to recognize the attacks they face, particularly under compromised skills, persistent state, and long-horizon execution attacks, and provide a systematic baseline for developing more reliable and secure agent systems.

- **核心问题**：现有智能体安全评估存在三大瓶颈：
  - **风险覆盖碎片化**：将提示注入、技能中毒、记忆污染等视为孤立威胁，缺乏统一抽象，难以诊断盲点。
  - **执行环境低耦合、低保真**：多依赖静态问答或模拟工具响应，无法再现多步骤调用、状态演变与系统级副作用，尤其无法评估状态持久的长周期攻击。
  - **评估指标单一、粗粒度**：通常仅依据最终回复或单次工具调用做二值判断，混淆了结果安全、风险意识与任务效用。
- **整体意义**：AgentCanary 旨在提供一个从风险建模、高保真执行到多维轨迹评估的端到端安全基准，以揭示智能体在真实可执行环境中的系统性安全缺陷。

### 2. 方法论
- **核心设计原则**：以“攻击者会瞄准的系统层面执行”为评估对象，将安全视为轨迹上环境后果的观察，而非仅看最终文字。
- **正交风险矩阵（Entry × Impact）**：
  - 将风险**入口**（e.g., 直接/间接提示注入、技能中毒、记忆污染、内在失效）与**影响**（e.g., 本地环境破坏、数据泄露、记忆污染、金融交易、网络攻击等7类）解耦。
  - 每个评测任务指定一个入口和一个目标影响，形成结构化的风险空间，使覆盖可审记且便于故障诊断。
- **高保真可执行环境**：
  - 基于 Docker 的沙箱环境，提供真实工具（文件系统、邮件、浏览器、银行系统、命令行等）而非模拟响应。
  - 任务工件自动化预置（网页、邮件线程、银行账户、技能文件等），确保环境状态真实可达。
  - 任务级隔离且支持单任务内状态持久，能够测评跨步骤、跨会话的长周期攻击。
  - 跨代理框架扩展接口，支持将同一任务套件在不同代理框架（如 OpenClaw、Hermes、NanoClaw）上评估。
- **轨迹驱动的三维评估**：
  - **轨迹定义**：$\Psi = (\xi_1,\dots,\xi_n)$ 包含用户查询、工具调用、工具返回及代理回复，并附系统级观测（文件修改、网络连接等）。
  - **三个正交分数**：
    - **结果安全分数（OSS）**：基于执行轨迹判定环境是否有实际损害（公式 $\sigma_{\text{out}} = J_{\text{out}}(\Psi; G_{\text{out}})$）。
    - **安全意识分数（SAS）**：评估智能体是否识别出风险源和恶意机制（公式 $\sigma_{\text{aware}} = J_{\text{aware}}(\Psi; G_{\text{aware}})$）。
    - **任务效用分数（TUS）**：衡量在保持安全的前提下完成合法任务的程度（公式 $\sigma_{\text{util}} = J_{\text{util}}(\Psi; G_{\text{util}})$）。
  - **聚合**：$S_{\text{total}} = \alpha \sigma_{\text{out}} + \beta \sigma_{\text{aware}} + \gamma \sigma_{\text{util}}$，权重按任务类型自适应设定（如直接注入无独立合法任务，则 $\gamma=0$）。
  - 评估器使用固定的大模型（Claude Opus 4.6）对轨迹进行 rubric 评分，降低人工干扰。

### 3. 实验设计
- **数据集与基准**：AgentCanary 包含 496 个种子任务，覆盖 5 个风险入口（DPI: 46; IPI: 181; MC: 104; SP: 110; IF: 55）和 7 个风险影响类别。
- **被评估模型与环境**：
  - **12 个代表性大模型**：4 个专有模型（GPT-5.4, Claude Sonnet 4.5, Claude Opus 4.6, Qwen3.6-Plus）与 8 个开源模型（DeepSeek-V4-Pro, GLM-4.7-Flash, GLM-5, Kimi-K2.5, MiniMax-M2.5, Qwen3.5 系列 35B/122B/397B）。
  - 主要结果基于 OpenClaw 代理框架；额外在 Hermes 和 NanoClaw 上做跨框架对比。
- **攻击方法设置**：
  - **DPI**：静态直接注入 + **动态攻击演化（DAE）**（攻击 LLM 根据反馈迭代优化提示）。
  - **LPA**：长周期渐进攻击，攻击跨多个会话逐步植入状态后触发（31 人工 + 46 自动生成攻击链）。
  - **IPI**：三种注入模板（Ignore, Important, InjecAgent），并区分有无任务上下文。
  - **SP**：原始毒化技能 vs 脚本伪装（危险逻辑隐藏在可执行脚本内）。
- **基线对比与防御评估**：考察了三个开源运行时安全防御（ClawKeeper, SecureClaw, Shield）在部分模型上的叠加效果。
- **评价指标**：除三维分数外，还报告不安全结果率（UOR）、安全意识率（SAR）、任务成功率（TSR）等阈值指标；通过 4 次重复运行验证评分稳定性。

- **实验规模**：
  - 12 个模型 × 5 主要风险入口（DPI 分 无/有 DAE，IPI 分 有/无上下文 + 3 模板，SP 分 原始/伪装，MC，IF，LPA），形成大量模型–设置组合。

- **充分性与公平性**：
  - 实验覆盖不同模型系列、规模、攻击强化策略和环境，能充分揭示安全能力的差异与系统性弱点。
  - 同一任务定义、同一 LLM 评判器、相同隔离执行环境，保证了公平比较。

### 6. 主要结论与发现
- **当前智能体安全能力极不均衡**：仅 Claude Opus 4.6 整体较稳健（总得分 83.9），绝大多数模型存在明显脆弱性，安全性与参数规模无简单对应。
- **风险因威胁面而异**：间接注入总体较易抵抗（平均 UOR ≈9%），而技能中毒（伪装后平均 UOR 达 62%）和长周期攻击（LPA，7/12 模型 UOR >50%）最具破坏力。
- **安全结果与安全意识经常解耦**：尤其在 IPI 设置中，许多模型能达到高 OSS 却显示极低 SAS，表现为“沉默服从”——未识别攻击但碰巧无害，这种假安全对未来攻击变体十分脆弱。
- **动态与长周期攻击显著放大风险**：DAE 可使某些模型的 DPI 总得分骤降近 26 点；脚本伪装使技能中毒安全评分普遍断崖式下滑（如 Claude Sonnet 下降超 50 点）。
- **效用与安全强相关**：安全最强的模型通常也最能完成合法任务，未出现“过度拒接”换取安全的现象。
- **框架选择影响安全测量**：同一模型在不同框架下总得分最大差 15.2 点，表明安全评估不能脱离运行框架。
- **运行时防御有一定帮助，但无法弥补模型级缺陷**：最好的防御方案加持的开源模型仍与最强未防御模型有近 20 分差距。

### 7. 优点
- **风险覆盖系统化**：正交“入口–影响”矩阵使风险设计有结构、可比较、可诊断，远超简单攻击列表式基准。
- **高保真执行环境**：真实工具、动态预置、任务级隔离和内部状态持久化为评估长周期、状态相关攻击提供了基础，避免了模拟环境的失真。
- **轨迹维度分解**：拆解结果安全、安全意识与任务效用，能够揭示“结果安全但不意识风险”“意识风险但未阻断”等隐蔽脆弱模式。
- **攻击多样化与自适应**：动态攻击演化和长周期渐进攻击使评估更贴近真实对抗，暴露静态基准遗漏的弱点。
- **可扩展与生态兼容**：支持即插即用不同框架和运行时防御，使基准具备部署级诊断价值。

### 8. 不足与局限
- **环境仍有限**：沙箱服务虽比模拟更真实，但与实际生产环境仍有差距，无法覆盖所有真实攻击面（如复杂网络拓扑、跨服务交互）。
- **评估依赖 LLM 裁判**：使用 Claude Opus 4.6 作为评分器，可能在特定场景引入偏差，尤其对安全意识这类较主观的维度。
- **运行时防御评估不全面**：仅测试三种防御组件，样本较少，且未与其他防御策略（如动态监控、行为审计）比较。
- **智能体多样性有限**：实验虽覆盖 12 模型和 3 框架，但日常应用有更多自研框架和定制化工具链，通用代表性可能存在缺口。
- **缺少真实成本考量**：未讨论安全机制带来的延迟或计算开销在真实部署中的可行性。
