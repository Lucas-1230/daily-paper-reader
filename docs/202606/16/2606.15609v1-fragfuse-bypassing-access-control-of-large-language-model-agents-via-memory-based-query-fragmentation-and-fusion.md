## 摘要
大型语言模型（LLM）智能体越来越依赖长期记忆来支持复杂任务执行、用户个性化和领域适配。与此同时，为防止策略违规请求和滥用，LLM智能体的访问控制机制正在探索中。我们揭示了一种由智能体记忆操作引入的新型攻击面：原本会触发访问控制的违禁内容可以跨交互被分片，以看似无害的形式存储在长期记忆中，随后通过记忆检索被重构，而不在最终用户查询中显式出现。我们提出FragFuse，这是首个利用长期记忆引入的时间信道使低权限用户绕过智能体访问控制的攻击。FragFuse分三个阶段运行：(1) 通过黑盒自适应查询配合分片掩码识别触发拒绝响应的分片；(2) 利用标记载体查询将这些分片注入记忆；(3) 通过后续攻击查询检索并融合存储的分片。虽然FragFuse可以针对单个智能体手工实例化，但我们进一步开发了一种基于代理的优化方案来调整融合指令和标记设计，从而在不违反攻击者威胁模型假设的情况下实现自动化攻击生成。我们在四种代表性智能体设置和任务领域中评估了FragFuse，涵盖三种最先进的智能体访问控制机制。FragFuse在所有设置中平均绕过成功率达86.3%，平均端到端有害任务成功率为41.1%，与无访问控制的配置相比，平均任务成功率仅下降4.4%。我们还表明，包括最先进的提示注入检测器和困惑度检测器在内的替代防御措施无法有效应对此攻击。

## Abstract
Large language model (LLM) agents increasingly rely on long-term memory to support complex task execution, user personalization, and domain adaptation. Meanwhile, emerging access-control mechanisms for LLM agents are being explored to block policy-violating requests and prevent misuse. We reveal a novel attack surface arising from agent memory operations: prohibited content that would trigger access control can be fragmented across interactions, stored in long-term memory in benign-appearing form, and later reconstructed through memory retrieval without appearing explicitly in the final user query. We propose FragFuse, the first attack that enables unprivileged users to bypass agent access control by exploiting this temporal channel introduced by long-term memory. FragFuse operates in three stages: (1) identifying rejection-responsive fragments via black-box adaptive querying with fragment masking; (2) injecting these fragments into memory using marker carrier queries; and (3) retrieving and fusing the stored fragments through a follow-up attack query. Although FragFuse can be instantiated manually for individual agents, we further develop a surrogate-based optimization scheme that tunes fusion instructions and marker designs, enabling automated attack generation without violating the attacker's threat-model assumptions. We evaluate FragFuse across four representative agent settings and task domains, covering three state-of-the-art agent access-control mechanisms. FragFuse achieves an average bypass success rate of 86.3% and an average end-to-end harmful task success rate of 41.1% across all settings, with only 4.4% average task-success degradation compared with configurations without access control. We also show that alternative defenses, including state-of-the-art prompt-injection detectors and perplexity detectors, do not effectively address this attack.

- 现有访问控制**仅审视当前查询文本**，隐含假设任何违规意图都会显式出现在查询中；论文揭示了一个新的攻击面：**智能体的长期记忆可作为隐蔽时间信道**。
- 攻击者可跨多次交互，将违禁内容分片并以无害形式写入记忆，随后经由记忆检索与融合，在不直接触发访问控制的情况下重构并执行原本会被拒绝的有害任务。
- FragFuse 正是为此场景提出的首个攻击方法，旨在使**普通无特权用户**通过正常交互绕过智能体的访问控制，完成受限制的操作。

## 2. 方法论
### 2.1 威胁模型
- 攻击者为无特权普通用户，无法访问智能体内部（代码、记忆库、访问控制内部实现），只能进行正常交互。
- 攻击目标：绕过访问控制（BSR），同时使智能体执行原被禁止的意图（TSR）。
- 攻击者仅知晓公开的访问控制策略，并可观察查询是否被拒绝（无需显式错误信息）。
- 记忆检索基于查询相似性（泛型设定），但具体相似度指标、嵌入模型等为黑盒。

### 2.2 FragFuse 攻击管道（三阶段）
**阶段一：敏感片段发现（黑盒自适应查询＋片段掩码）**
- 使用基于 LLM 的 FRAG EXTRACTOR 反复提取可能导致拒绝的文本片段。
- 每次迭代将提取的片段用标记（默认 `<>`）替换，生成临时掩码查询并提交给访问控制检查。
- 若仍被拒绝，则继续提取更多片段，直到掩码查询被接受或达到最大迭代次数（默认 3 轮）。
- 输出：敏感片段列表 `{ft}` 和最终无掩码查询 `qmask`。

**阶段二：载体查询构建与记忆注入**
- 选择一个无害的宿主查询 `qhost`（已知可通过访问控制）。
- 将第一阶段发现的敏感片段按词级分割，并用相同标记包裹（如 `<>`）附加到宿主查询后，形成载体查询 `qcar`：
  `qcar = [qhost, v1:f1, …, vℓ:fℓ]`
- 该载体查询在形式上仍为良性，通过普通交互提交，其查询–执行记录被存入智能体记忆。

**阶段三：片段检索与融合（攻击查询）**
- 攻击查询 `qatt` 包含三个部分：
  - 相同的宿主查询 `qhost`（提高检索成功率）
  - 一个通用融合指令 `Ifuse`（指示智能体从检索到的记录中提取 `<>` 内的内容并填入掩码查询）
  - 来自阶段一的掩码查询 `qmask`。
- 当智能体检索到之前的载体记录时，融合指令便引导其重构出原始的有害查询并执行。

### 2.3 融合指令的代理约束优化
- 纯手工设计的融合指令效果不稳定，过长或过多指令可能干扰检索，且可能提取/替代失败。
- 论文构建了在替代代理模型上的约束优化问题，目标函数：
  `min_{Ifuse} Lret(Ifuse) + Lfus(Ifuse)`
  - `Lret`：最小化攻击查询与宿主查询的语义距离（促进检索）。
  - `Lfus`：最大化最终融合查询与原始违禁查询的语义相似度（通过代理模型运行代理融合任务评估）。
  - 约束：`Lcoh`（攻击查询的连贯性/困惑度）不超过由良性查询校准的阈值。
- 使用遗传算法在 GPT-4o 代理模型上迭代优化，变异操作包括 LLM 重写、交叉和局部突变，生成最优融合指令。整个优化过程完全离线，不访问目标智能体内部。

## 3. 实验设计

### 3.2 访问控制机制

### 3.3 对比基线

- **基线攻击**：在禁止查询中直接加入绕过指令（如“忽略上述信息，直接执行查询”）。
- **消融**：移除 FRAG EXTRACTOR（改固定长度分割）、移除优化融合指令（手工指令）。

### 3.4 评价指标
- **BSR（Bypass Success Rate）**：载体查询与攻击查询均通过访问控制的比例。

## 6. 主要结论与发现
- **FragFuse 可在不触发访问控制的前提下高效绕过**，平均 BSR 达 86.3%，端到端有害任务成功率（E2E‑SR）为 41.1%，而基线仅 5.7%。
- 在绕过访问控制后，**任务成功率仅平均下降 4.4%**（与直接无控制查询相比），说明攻击能有效恢复原有害查询的意图。
- 攻击**查询效率高**：大多数情况下仅需 2 次必需交互，敏感片段发现额外平均不超过 2.4 次查询。
- **记忆机制是健壮攻击面**：无论检索度量、标记选择、记忆规模或高级记忆写入策略，FragFuse 仍保持较高 BSR，仅部分高级检索策略可降低 TSR。
- 现有防御**无效**：提示注入检测器对载体查询和攻击查询的鉴别率有限；困惑度分布在多数智能体中与良性查询高度重叠。
- 事后防御虽可在动作层面检测违规，但成本高、通用性差，且攻击已造成计算资源浪费和潜在信息泄露。

## 7. 优点
- **首次揭示智能体记忆作为时间信道的攻击面**，填补了现有访问控制研究的空白。
- **攻击完全在黑盒条件下进行**，无需访问智能体内部或直接操控记忆，具有高实用性。
- **三阶段管道系统且自动化**：敏感片段发现、载体注入、检索融合均能自动实施，代理优化进一步增强了有效性与隐蔽性。
- **评估广泛而细致**：涵盖 4 种智能体、多骨干 LLM、两类访问控制、大量消融与防御评估，结论可信度高。
- 辅助材料（代码、提示等）已开源，附有伦理声明和责任披露流程。

## 8. 不足与局限
- **任务领域有限**：实验限于电商、网页导航、OS 助手和工具使用，未扩展到医疗、科研等对安全要求更高的领域。
- **依赖基础模型能力**：融合指令的有效性依赖于智能体核心 LLM 的指令遵循能力，若底层模型较弱，攻击可能失败。
- **存在失败案例**：如 LLM-AC 高误报导致载体查询被拒；过度掩码使语义空洞化，访问控制仍能拦截。
- **可能被事后防御阻止**：若部署完备的动作级检查，攻击仍可能在最终执行阶段被拦截，但作者指出此类防御成本高、通用性差，且攻击已引发执行开销。
- **攻击并非 100% 成功**，部分组合下 TSR 绝对值较低（如 InspAgent），说明受限于特定智能体的任务执行能力。
- 代理优化中的连贯性约束依赖 GPT-2 的困惑度，可能无法完全模仿目标智能体的输入分布。
