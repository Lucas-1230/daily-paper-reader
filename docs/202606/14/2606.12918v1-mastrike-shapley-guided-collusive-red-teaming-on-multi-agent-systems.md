## 摘要
分层多智能体系统（MAS）正迅速部署于金融和软件工程等领域的高风险工作流程中。在这些系统中，安全性和防护天然分布于角色专精的智能体之间，显著扩展了攻击面，尤其是在权限提升和跨智能体合谋等协同对抗行为下。现有的MAS红队测试方法仍有局限：它们依赖启发式选择目标智能体并扰动孤立的消息流，未能回答关键问题，如哪些智能体对系统安全最为重要，以及受感染的智能体如何协同绕过防御。我们提出MAStrike，一个用于分层MAS合谋红队测试的闭环框架。我们首次提出针对MAS的智能体级Shapley值分析，量化在特定任务分布下每个智能体对系统鲁棒性的边际贡献。在此归因的指导下，MAStrike识别脆弱的智能体联盟并生成协同的、角色感知的对抗操作。这些攻击通过结构化的因果诊断迭代优化，将失败案例归因于阻止对抗尝试的未受感染智能体。我们进一步构建了一个全面的MAS红队测试基准和可控环境，涵盖多种分层拓扑和领域，包括金融、软件工程和CRM。基于多个前沿模型构建的MAS上的广泛实验表明，MAStrike显著优于启发式基线。我们的分析进一步揭示了智能体间非平凡的Shapley值分布和高阶交互结构，暴露了先前单智能体或基于模板的方法所忽视的关键漏洞与协同模式。

## Abstract
Hierarchical multi-agent systems (MAS) are rapidly being deployed in high-stakes workflows across domains such as finance and software engineering. In these systems, safety and security are inherently distributed across role-specialized agents, significantly expanding the attack surface, particularly under coordinated adversarial behaviors such as privilege escalation and cross-agent collusion. Existing red-teaming approaches for MAS remain limited: they rely on heuristic selection of target agents and perturb isolated message streams, leaving critical questions unanswered as which agents are most responsible for system safety, and how compromised agents can coordinate to bypass defenses. We propose MAStrike, a closed-loop framework for collusive red-teaming in hierarchical MAS. We propose the first agent-level Shapley value analysis for MAS, quantifying each agent's marginal contribution to system robustness under task-specific distributions. GGuided by this attribution, MAStrike identifies vulnerable agent coalitions and generates coordinated, role-aware adversarial manipulations. These attacks are iteratively refined through structured causal diagnosis, attributing failure cases to uncompromised agents that block adversarial attempts. We further build a comprehensive MAS red-teaming benchmark and controllable environments spanning diverse hierarchical topologies and domains, including finance, software engineering, and CRM. Extensive experiments across MAS built on multiple frontier models show that MAStrike substantially outperforms heuristic baselines. Our analysis further uncovers non-trivial Shapley value distributions and higher-order interaction structures among agents, revealing critical vulnerabilities and coordination patterns that are overlooked by prior single-agent or template-based methods.

## 2. 方法论

### 2.1 整体架构
MAStrike 包括两个核心阶段：  
1. **智能体级 Shapley 值分析**：量化各智能体对系统安全的边际贡献及智能体间的协同效应。  
2. **Shapley 引导的合谋攻击优化**：基于 Shapley 归因选择攻击联盟，生成协同对抗注入，并通过失败诊断迭代优化。

### 2.2 智能体级 Shapley 值分析
- **任务生成**：根据 MAS 拓扑和智能体角色设计多样化的恶意红队任务（覆盖不同工作流），作为估计攻击成功率的基础。
- **联盟攻击成功率估计**：通过采样不同联盟（子集），在受感染智能体上覆盖输出为一致且良性外观的对抗响应，其余智能体正常行为，估计联盟攻击成功率 \(v_q(C) = \text{ASR}_q(C)\)。
- **Shapley 值计算**：  
  - 单智能体边际贡献：  
    \[
    \phi_i(q) = \sum_{C \subseteq A \setminus \{a_i\}} \frac{|C|!(|A|-|C|-1)!}{|A|!} [v_q(C \cup \{a_i\}) - v_q(C)]
    \]
  - 两两交互指数（Shapley interaction index）：  
    \[
    I_{ij}(q) = \sum_{C \subseteq A \setminus \{a_i,a_j\}} \frac{|C|!(|A|-|C|-2)!}{(|A|-1)!} [v_q(C \cup \{a_i,a_j\}) - v_q(C \cup \{a_i\}) - v_q(C \cup \{a_j\}) + v_q(C)]
    \]
  - 为降低计算复杂度，对较大智能体集采用分层抽样估计，小联盟优先。
- **归因迁移**：对目标任务 \(q\)，基于 KNN 检索相似采样任务，用 softmax 权重聚合其 Shapley 值和交互指数，得到估计的 \(\hat{\phi}_i(q)\) 和 \(\hat{I}_{ij}(q)\)。

### 2.3 联盟选择与合谋攻击生成
- **协同感知联盟选择**：在预算 \(k\) 下，最大化联合目标函数：  
  \[
  C_k^\star(q) = \arg\max_{C \subseteq A, |C|=k} \left( \sum_{a_i \in C} \hat{\phi}_i(q) + \sum_{\{a_i,a_j\} \subseteq C} \hat{I}_{ij}(q) \right)
  \]
- **迭代优化**：  
  - 红队智能体基于选定联盟，**联合生成**所有感染智能体的对抗操纵（提示注入、工具操纵、环境注入等），确保跨智能体信息一致。  
  - 在完整 MAS 上执行攻击，收集执行轨迹。  
  - 设计评估函数判断攻击是否成功，并提供结构化反馈。  
  - 若失败，进行**结构化因果诊断**，识别阻止攻击的未受感染智能体，并细化攻击注入，形成生成-执行-评估-细化的闭环。

### 2.4 威胁模型
- 攻击者了解 MAS 结构、智能体角色、通信拓扑及输入输出消息，但无法访问模型参数。
- 可感染除目标工具智能体外的任意 \(k\) 个智能体，通过注入等方式操纵其行为，并允许合谋。

## 6. 主要结论与发现
- **MAStrike 显著优于所有基线**：在 \(k=2\) 下，平均 ASR 达到 55.6%–61.8%，而基线大多接近 0%。尤其在工程和金融领域可达到 70%–100%。
- **Shapley 值揭示智能体重要性稀疏且任务依赖**：少数智能体对安全起决定性作用，高影响智能体因任务而异。
- **交互模式高度任务相关**：强正交互集中在特定智能体对，单独重要性高不代表联盟协同强，需联合建模。
- **MAStrike 的联盟选择和协同生成使攻击随 \(k\) 稳定提升**，而基线方法可能因不一致消息导致 ASR 下降。
- **企业级护栏在 MAS 攻击下存在检测盲区**：合谋生成的协调信号使对抗执行轨迹与良性轨迹难以区分，且检测效果在联盟和风险类别间差异大。

## 7. 优点
- **首度将 Shapley 值应用于 MAS 安全归因**：量化智能体边际贡献与交互效应，为攻击联盟选择提供理论依据。
- **闭环协同攻击框架**：联合生成 + 失败诊断 + 迭代细化，能自适应地突破分布式安全校验。
- **构建了高质量、可复现的多域 MAS 基准**：覆盖金融、工程、CRM，环境可控，任务设计强调合谋。
- **实验全面**：多模型、多基线、多预算、迁移实验，并分析了 Shapley 分布与交互结构。
- **实用启发**：指出单独重要性不等于协同强度，为 MAS 防御设计提供了新视角。
